# 版本管理

本文档定义「光轨逆行」方案的版本命名规则和变更记录管理规范。

---

## 一、版本命名规则

### 命名格式

```
GGXN_[交付物]_v[主版本].[次版本].[修订号]_[日期]
```

### 各字段说明

| 字段 | 说明 | 取值范围 | 示例 |
|------|------|----------|------|
| GGXN | 方案缩写（光轨逆行拼音首字母） | 固定值 | GGXN |
| 交付物 | 交付物类型标识 | powerbank_front / powerbank_back / cabinet_front | powerbank_front |
| 主版本 | 重大设计变更 | 1, 2, 3... | v1 |
| 次版本 | 功能性调整（新增/移除设计元素） | 0, 1, 2... | v1.0 |
| 修订号 | 小幅优化（色彩微调、位置微调等） | 0, 1, 2... | v1.0.0 |
| 日期 | 版本日期（可选，用于归档） | YYYYMMDD | 20260415 |

### 版本号递增规则

| 变更类型 | 递增字段 | 示例 |
|----------|----------|------|
| 重大设计变更（整体视觉方向调整、核心元素替换） | 主版本 | v1.0.0 → v2.0.0 |
| 功能性调整（新增/移除设计元素、布局重组） | 次版本 | v1.0.0 → v1.1.0 |
| 小幅优化（色彩微调、位置微调、细节修正） | 修订号 | v1.0.0 → v1.0.1 |

### 完整文件名示例

```
GGXN_powerbank_front_v1.0.0_20260415.png
GGXN_powerbank_back_v1.0.0_20260415.png
GGXN_cabinet_front_v1.0.0_20260415.png
GGXN_powerbank_front_v1.1.0_20260420.png
GGXN_powerbank_front_v1.0.1_20260418.png
```

## 三、文件归档规范

### 目录结构

```
versions/
├── 版本管理.md                    # 本文件
├── v1.0.0/                        # 版本 v1.0.0 归档
│   ├── 源文件/
│   │   ├── GGXN_powerbank_front_v1.0.0.ai
│   │   ├── GGXN_powerbank_back_v1.0.0.ai
│   │   └── GGXN_cabinet_front_v1.0.0.ai
│   ├── 预览文件/
│   │   ├── GGXN_powerbank_front_v1.0.0_preview.png
│   │   ├── GGXN_powerbank_back_v1.0.0_preview.png
│   │   └── GGXN_cabinet_front_v1.0.0_preview.png
│   └── 变更说明.md
├── v1.1.0/                        # 版本 v1.1.0 归档
│   └── ...
└── 变更日志.md                    # 全局变更日志
```

### 归档要求

1. **源文件归档**：每个版本的源文件（.ai 或 .psd）需归档至对应版本目录下的 `源文件/` 文件夹
2. **预览文件归档**：每个版本的 RGB 预览图需归档至对应版本目录下的 `预览文件/` 文件夹
3. **变更说明**：每个版本需附带 `变更说明.md`，记录本版本的具体变更内容
4. **全局变更日志**：所有版本的变更记录需汇总至 `变更日志.md`

## 六、版本管理注意事项

1. **禁止覆盖**：任何已有版本的文件不得直接覆盖，新版本必须使用新的版本号
2. **命名一致**：源文件、预览文件、交付物的文件名中的版本号必须一致
3. **同步更新**：outputs/ 目录下的交付物文件应始终为最新版本，历史版本归档至 versions/
4. **变更记录**：每次版本更新必须同步更新变更日志，确保变更历史可追溯
5. **源文件保留**：所有版本的源文件必须保留，不可删除历史版本的源文件
