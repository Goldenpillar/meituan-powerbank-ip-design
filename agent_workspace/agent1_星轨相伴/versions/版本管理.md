# 版本管理

> 本文档定义「星轨相伴」方案的版本命名规则、版本递进规范和变更记录模板。

---

## 一、版本命名规则

### 1.1 命名格式

```
XGCB_[交付物]_v[主版本].[次版本].[修订号]_[日期]
```

### 1.2 各字段说明

| 字段 | 说明 | 取值范围 | 示例 |
|------|------|----------|------|
| XGCB | 方案代号（星轨相伴拼音首字母） | 固定值 | XGCB |
| [交付物] | 交付物标识 | 见下方交付物标识表 | powerbank_front |
| v | 版本前缀 | 固定值 | v |
| [主版本] | 主版本号 | 1-99 | 1 |
| [次版本] | 次版本号 | 0-99 | 0 |
| [修订号] | 修订号 | 0-99 | 0 |
| [日期] | 日期（可选） | YYYYMMDD | 20260415 |

### 1.3 交付物标识表

| 交付物标识 | 对应交付物 | 存放路径 |
|-----------|-----------|----------|
| powerbank_front | 充电宝正面设计稿 | outputs/powerbank/ |
| powerbank_back | 充电宝背面设计稿 | outputs/powerbank/ |
| cabinet_front | 机柜正面设计稿 | outputs/cabinet/ |

### 1.4 命名示例

```
XGCB_powerbank_front_v1.0.0.png          # 充电宝正面 v1.0.0
XGCB_powerbank_front_v1.0.0_20260415.png # 充电宝正面 v1.0.0（带日期）
XGCB_powerbank_back_v1.0.0.png           # 充电宝背面 v1.0.0
XGCB_cabinet_front_v1.0.0.png            # 机柜正面 v1.0.0
XGCB_powerbank_front_v1.1.0.png          # 充电宝正面 v1.1.0（次版本更新）
XGCB_powerbank_front_v2.0.0.png          # 充电宝正面 v2.0.0（主版本更新）
```

## 二、版本递进规范

### 2.1 主版本号（Major）— 递增条件

主版本号在以下情况下递增：

- 设计方案发生重大变更（如整体布局调整、色彩方案更换）
- 品牌元素位置或尺寸发生变更
- 印刷尺寸或工艺方案发生变更
- 经过正式评审通过的设计定稿

**示例**：v1.0.0 → v2.0.0

### 2.2 次版本号（Minor）— 递增条件

次版本号在以下情况下递增：

- 新增设计元素（如增加星轨层数、调整星点分布）
- 修改渐变背景的色彩参数
- 调整夜光油墨区域的范围
- 根据内部评审意见进行的设计优化

**示例**：v1.0.0 → v1.1.0

### 2.3 修订号（Patch）— 递增条件

修订号在以下情况下递增：

- 修正设计错误（如对齐偏差、色彩偏差）
- 调整工艺标注（如夜光油墨区域边界微调）
- 修正文字错误或标注错误
- 文件格式转换或导出参数调整

**示例**：v1.0.0 → v1.0.1

### 2.4 版本递进规则总结

| 变更类型 | 递增字段 | 示例 |
|----------|---------|------|
| 重大方案变更 | 主版本号 +1，次版本号和修订号归零 | v1.3.2 → v2.0.0 |
| 功能性优化 | 次版本号 +1，修订号归零 | v1.3.2 → v1.4.0 |
| 错误修正 | 修订号 +1 | v1.3.2 → v1.3.3 |

## 三、文件管理规范

### 3.1 文件命名

- 最终交付文件使用上述命名规则
- 源文件（AI/PSD）使用相同的版本号命名，仅扩展名不同
- 工作中的草稿文件可在文件名中添加 `_draft` 或 `_WIP` 后缀

```
XGCB_powerbank_front_v1.0.0.ai           # AI源文件
XGCB_powerbank_front_v1.0.0.psd          # PSD源文件
XGCB_powerbank_front_v1.0.0.png          # PNG交付文件
XGCB_powerbank_front_v1.1.0_WIP.png      # 工作中的草稿
```

### 3.2 文件存放

- 所有版本的最终交付文件存放在对应的 `outputs/` 子目录中
- 历史版本文件保留，不删除或覆盖
- 草稿文件可存放在 `outputs/` 对应目录的 `_drafts/` 子目录中

```
outputs/
├── powerbank/
│   ├── _drafts/                              # 草稿文件
│   │   ├── XGCB_powerbank_front_v1.1.0_WIP.png
│   │   └── ...
│   ├── XGCB_powerbank_front_v1.0.0.png      # v1.0.0 交付
│   ├── XGCB_powerbank_front_v1.0.0.ai       # v1.0.0 源文件
│   ├── XGCB_powerbank_back_v1.0.0.png       # v1.0.0 交付
│   └── ...
└── cabinet/
    ├── _drafts/
    ├── XGCB_cabinet_front_v1.0.0.png
    └── ...
```

### 3.3 日期后缀使用规范

- 日期后缀 `[YYYYMMDD]` 为可选项
- 建议在以下场景使用日期后缀：
  - 向外部（客户、印刷厂）发送文件时
  - 需要追溯特定日期的设计状态时
  - 同一天内产生多个修订版本时

## 五、协作规范

### 5.1 分支管理

如多人协作，建议采用以下分支策略：

- `main` 分支：存放已评审通过的正式版本
- `dev` 分支：存放开发中的版本
- `review` 分支：存放待评审的版本

### 5.2 评审流程

1. 设计完成后，将文件提交至 `review` 分支
2. 发起评审请求，附上变更说明
3. 评审通过后，合并至 `main` 分支，更新版本号
4. 评审未通过，返回 `dev` 分支继续修改

### 5.3 文件备份

- 每个主版本发布后，打包备份至独立目录
- 备份文件命名：`XGCB_backup_v[主版本]_[日期].zip`
- 备份内容包含：所有源文件、交付文件、素材文件
