## 快速开始

### 1. 素材准备

将设计素材放入对应目录：

```bash
# 方案一素材
assets/方案一_星轨相伴/

# 方案二素材
assets/方案二_光轨逆行/

# 品牌素材（需从品牌部门获取）
assets/品牌素材/
```

### 2. 版本管理

每次设计更新，创建新版本目录：

```bash
# 创建新版本
versions/方案一_星轨相伴/v1.0.0/

# 版本命名规则
[方案代码]_[交付物]_v[主版本].[次版本].[修订号]_[日期]
```

### 3. 参照查阅

设计过程中参考以下目录：

```bash
# 竞品分析
references/竞品分析/

# 行业趋势
references/行业趋势/

# 技术文档
references/技术文档/
```

### 4. 最终输出

完成设计后，将最终文件放入：

```bash
outputs/方案一/
outputs/方案二/
```

---

## 核心原则

1. **落地可实施性为第一优先级**
2. **素材可独立使用、可组合**
3. **版本管理规范，便于追溯**
4. **参照系统完善，避免雷同**
